## Summary

AI assistance (Claude) was used throughout this project as a **drafting and explanation tool** — for generating starting structures for SQL objects, ASP.NET Core boilerplate, and debugging error messages. Every piece of generated code was tested in SSMS or run locally before being committed, and concepts (normalization, RLS, triggers, transactions, window functions) were reviewed and explained in my own words as part of the development process, not just copied. Several real implementation problems (the cascade-path SQL error, the Razor `section` naming conflict, the EF Core migration conflict with manually-created tables, and the local-vs-cloud network connectivity issue) required independent debugging and decision-making beyond what AI initially suggested, and are documented above and in the project report's "Challenges" section.
